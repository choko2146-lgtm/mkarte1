# 04. 撮影・保存フロー

## 1. 基本方針

写真は、撮影後に顧客を選択して紐づける。

撮影直後は一時保存し、顧客選択後に正式保存する。

## 2. 一時保存

撮影直後のファイル名：

```text
TEMP_yyyyMMdd_HHmmss.jpg
```

例：

```text
TEMP_20260516_143000.jpg
```

## 3. 正式保存

顧客選択後のファイル名：

```text
yyyyMMdd_顧客名.jpg
```

例：

```text
20260516_山田太郎.jpg
```

## 4. 同名ファイルがある場合

連番を付ける。

```text
20260516_山田太郎.jpg
20260516_山田太郎_01.jpg
20260516_山田太郎_02.jpg
```

## 5. 保存先

```text
Pictures/おかんのカルテ/顧客名/
```

## 6. ファイル名禁止文字

以下の文字は `_` に置換する。

```text
/ \ : * ? " < > |
```

## 7. 撮影フロー

1. ホーム画面でカメラを押す
2. 撮影画面を表示
3. 撮影ボタンを押す
4. 権限確認
5. カメラ起動
6. 写真撮影
7. 一時保存
8. 撮影後顧客選択画面を表示
9. 顧客検索
10. 顧客選択
11. 未登録なら新規登録
12. 写真と顧客を紐づけ
13. 正式ファイル名生成
14. 正式保存
15. Photo EntityをRoomに保存
16. 完了メッセージ表示

## 8. 写真削除仕様

写真削除時は以下を両方削除する。

- Room Database上のPhoto情報
- 端末内の写真ファイル

削除前に確認ダイアログを表示する。

```text
この写真を削除しますか？
端末内の画像も削除されます。
```
